git常用命令总结

  • git

创建操作

  • mkdir (name) 创建目录
  • cd (name) 进入到该目录
  • pwd 查看当前目录路径
  • git init 将目录变成Git管理仓库
  • git add name 将库中文件添加到暂存区
  • git add -a 将库中所有文件添加到暂存区
  • git commit -am “提示信息” 将文件提交到仓库

回退操作

  • git diff 查看修改
  • git log 查看历史记录

回退操作

  • git reset –hard HEAD ^ 回退上一个版本
  • git reset –hard HEAD ^^ 回退两个版本
  • git reset –hard 版本号 回退到指定版本
  • git reflog 记录每条命令
  • cat 文件名 查看文件内容

撤销操作

  • git checkout –<文件名> 撤销还未添加到暂存区的修改
  • git reset HEAD <文件名> 撤销已经添加到暂存区的修改
  • git rm <文件名> 删除文件

分支操作

分支简单操作

  • git branch (git branch -a) 查看分支
  • git branch <分支名> 创建分支
  • git checkout <分支名> 切换分支
  • git checkout -b <分支名> 创建并切换分支
  • git merge <分支名> 合并某分支到当前分支
  • git branch -d <分支名> 删除分支
  • git branch -D <分支名> 强制删除一个未合并分支
  • 合并冲突： 当两条分支对同一文件的同一文本进行不同的修改并试图合并时，Git不能自动合并的，称之为冲突（conflict），解决冲突需要人工处理。对于简单的合并，手工编辑，然后去掉这些标记，最后像往常的提交一样先add再commit即可。
  • 分支管理：通常合并分支，Git会用Fast forward模式，但这种模式下，删除分支后会丢掉分支信息，使用–no–ff会保留信息。使用–no–diff会创建一个新的commit，所以要带上-m参数 git merge –no–ff “commit message” 分支名
  • bug分支：离开当前分支并保存当前工作区
  • git stash 存储当前工作区
  • git stash list 查看存储的工作区
  • git stash apply 恢复删除之前保留的工作区
  • git stash drop 删除工作区内容
  • git stash pop 恢复保留的工作区，并删除内容

远程分支

  • git remote -v 查看远程分支
  • git push origin <分支名> 本地推送分支到远程库
  • git checkout -b 本地分支名 origin/远程分支名 本地创建和远程对应分支
  • git branch –set upstream 本地分支名 origin/远程分支名 建立本地分支和远程分支的关联

分支标签操作

  • git tag
  • git tag name 创建标签
  • git tag -a name commit的id 给制定commit创建标签
  • git tag -d name 删除本地标签
  • git push origin name 推送一个本地标签
  • git push origin –tags 推送所用标签
  • git pish origin :refs/tags/远程标签名 删除一个远程标签
